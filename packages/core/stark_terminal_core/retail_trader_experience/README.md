# Retail Trader Experience

The `retail_trader_experience` package is planning and guardrails only in
Prompt 56.

It defines:

- planning contracts
- persona placeholders
- journey placeholders
- experience section placeholders
- experience card placeholders
- dashboard, decision, and safety reference placeholders
- forbidden interaction contracts
- safety policy and result helpers
- readiness report templates
- health metadata

It does not create active UI, frontend components, desktop components,
recommendation cards, action cards, confidence score displays, active
DecisionObject displays, readiness-to-trade displays, broker controls,
execution APIs, or suitability profiling.

Future prompts may add API and display contract skeletons only after audit
checks confirm the planning boundary remains intact.
