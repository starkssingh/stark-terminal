# Retail Trader Experience Card Placeholders

Prompt 56 defines experience card placeholder contracts for future Retail
Trader Experience planning.

## Card Schema

Card placeholders include an ID, card kind, title, description, visibility,
planning-only flag, unavailable flag, safety label, and explicit blocked
display/control flags.

Planned card kinds include generic, trader context, instrument context,
dashboard context, data quality, safety, educational, review, and unavailable
placeholders.

## Forbidden Interpretation

Cards are not recommendation cards, action cards, confidence score cards,
DecisionObject display cards, readiness-to-trade cards, broker-control cards,
suitability profile cards, approval/override cards, execution cards, active UI,
frontend components, or desktop components.

Development remains Mac mini M2 / macOS / Apple Silicon. The target desktop
product remains Windows-native Stark Terminal.
