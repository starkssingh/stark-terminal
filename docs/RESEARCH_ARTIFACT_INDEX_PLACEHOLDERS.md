# Research Artifact Index Placeholders

Research Artifact Index placeholders are planning-only contracts. They reserve metadata shapes for a future index without creating an indexing engine, search engine, ranking engine, retrieval engine, embedding pipeline, vector store, persistent storage, ingestion, file preview, paper parsing, strategy generation, backtesting, recommendations, or execution.

## Placeholder Families

- Metadata placeholders describe future index records.
- Key placeholders describe future artifact, registry, reference, provenance, tag, category, source-label, and lifecycle keys.
- Reference placeholders describe future links to artifact, registry, provenance, and lifecycle references.
- Tag placeholders describe future topic, asset-class, market, method, dataset, paper-reference, experiment, status, and safety labels.
- Provenance placeholders describe future audit notes and source references without validating source trust.
- Lifecycle placeholders describe safe statuses only: PLACEHOLDER, REFERENCED, DRAFT, REVIEW_REQUIRED, BLOCKED, DEFERRED, UNKNOWN.

All placeholders remain unavailable-by-default and cannot be treated as indexed, searchable, ranked, embedded, retrieved, validated, recommended, ready-to-trade, or executable.
