# Retail Trader Experience Journey Placeholders

Prompt 56 defines Retail Trader Experience journey placeholder contracts.

## Journey Schema

Journey placeholders include an ID, journey kind, title, description,
planning-only flag, unavailable flag, and explicit forbidden journey flags.
They are not rendered journeys and do not create active UI.

Planned journey kinds include onboarding, instrument review, dashboard context
review, safety context review, evidence review, human review, and unavailable
notice placeholders.

## Forbidden Interpretation

Journey placeholders cannot become trading advice journeys, recommendation
journeys, broker-control journeys, execution journeys, readiness-to-trade
journeys, approval journeys, override journeys, frontend components, desktop
components, or active decision screens.

Development remains Mac mini M2 / macOS / Apple Silicon. The target desktop
product remains Windows-native Stark Terminal.
