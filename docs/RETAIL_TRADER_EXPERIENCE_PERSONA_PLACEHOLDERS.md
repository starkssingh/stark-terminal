# Retail Trader Experience Persona Placeholders

Prompt 56 defines persona placeholder contracts for future Retail Trader
Experience planning.

## Persona Schema

Persona placeholders include an ID, persona kind, name, description, safety
label, notes, and fail-closed flags. They are planning-only records and are not
active profiles.

Planned persona kinds include retail trader, intraday trader, swing trader,
options context, risk-aware beginner, and quant-curious trader placeholders.

## Forbidden Interpretation

Persona placeholders must not become suitability profiling, trading permission
profiles, active personalization, recommendation behavior, action generation,
confidence scoring, DecisionObject generation, readiness-to-trade, broker
controls, or execution behavior.

Development remains Mac mini M2 / macOS / Apple Silicon. The target desktop
product remains Windows-native Stark Terminal.
