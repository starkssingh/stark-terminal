# Local Sample Provider Adapter

Prompt 21 implements Local Sample Provider Adapter v0 as the first concrete provider adapter. It is synthetic, local-only, test/dev only, read-only, and guardrail-protected.

This adapter is not a real provider. It makes no external calls and does not call NSE, BSE, broker, vendor, or web APIs. It performs no scraping, loads no credentials, ingests no real market data, writes no persistence stores, publishes no events, computes no analytics, generates no trading signals, generates no decisions, and exposes no execution APIs.

## Purpose

The adapter gives Stark Terminal a deterministic provider-shaped contract for tests and local development. It uses the existing synthetic instrument master and synthetic OHLCV fixture generation so downstream request/response validation can be exercised before any real provider integration exists.

Supported capabilities:

- instrument master
- historical bars
- health check

Unsupported capabilities:

- real latest bar
- real options chain
- real futures chain
- corporate actions
- broker execution
- order placement
- credentials or provider account access

## Guardrails

`LocalSampleProviderAdapter` evaluates the Prompt 20 provider guardrails before use. The only allowed mode is `SYNTHETIC_ONLY`. Network calls, scraping, credentials, real ingestion, broker behavior, live trading, real-money routing, and execution APIs remain blocked.

The adapter reports `LOCAL_SAMPLE` provider identity and uses `synthetic-local-test-only` source references. Every response is labelled synthetic/local/test-only and must not be presented as live, tradable, production, provider-sourced, or investment data.

## Data Quality

Instrument master and historical bar responses use `MarketDataResponse` contracts. Historical bars are generated by `SyntheticOHLCVConfig` and validated through the Data Quality Framework where practical before they are returned.

The adapter does not persist bars. Prompt 18 storage and Prompt 19 export services remain separate explicit workflows.

Development environment: Mac mini M2 / macOS / Apple Silicon. Target desktop product: Windows-native Stark Terminal.
