# Retail Dashboard Layout Placeholders

Retail Dashboard Display layout placeholders describe possible future screen compositions without rendering any UI. They are display contract skeleton artifacts only and are unavailable by default.

Layout kinds:

- desktop shell placeholder
- retail overview placeholder
- instrument detail placeholder
- risk context placeholder
- safety placeholder
- unavailable

No layout placeholder is an active dashboard. No layout placeholder is rendered now. No layout placeholder creates a frontend component or desktop UI component. No layout placeholder contains recommendation cards, action generation, confidence scoring, DecisionObject generation, readiness-to-trade, broker controls, approval controls, override controls, or execution APIs.

The layout placeholders are for future contract alignment only. They cannot be used as a broker-control layout, execution layout, or production-ready dashboard display.

Development environment: Mac mini M2 / macOS / Apple Silicon.
Target desktop product: Windows-native Stark Terminal.
