# Research Artifact Registry Display Metadata Placeholders

Prompt 72 display metadata placeholders describe how future artifact metadata may be referenced by a display contract. They are backend metadata contracts only.

Metadata placeholders cannot be treated as rendered UI, validated artifact records, active registry storage, or trusted research output.

Forbidden behavior remains:

- No active UI, frontend components, desktop components, active widgets, or rendered layouts.
- No active artifact ingestion, persistent storage, database tables, migrations, object storage, file upload, file download, or file preview.
- No paper ingestion, paper parsing, PDF parsing, arXiv ingestion, LLM paper analysis, method extraction, or strategy extraction.
- No strategy generation, strategy code generation, backtesting, optimization, recommendations, action generation, confidence scoring, DecisionObject generation, readiness-to-trade, broker controls, or execution APIs.

Metadata placeholders are read-only and unavailable-by-default. Future prompts and audits are required before any implementation can move beyond placeholder metadata.
