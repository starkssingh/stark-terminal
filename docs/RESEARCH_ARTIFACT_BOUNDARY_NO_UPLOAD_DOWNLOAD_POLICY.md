# Research Artifact Boundary No Upload Download Policy

Prompt 75 boundary hardening keeps file upload, file download, and file preview
forbidden. Policy keyword: no file upload/download and no file preview.

There are no file upload endpoints, no file download endpoints, no file
preview endpoints, no file byte handling, no local file reads, no arbitrary
file read API, and no external downloads. Reference placeholders remain
descriptive and are not fetched, read, validated, uploaded, downloaded, or
previewed.

Future prompt and audit required before unlocking any file handling. No active
ingestion/storage, active UI, paper parsing, strategy
generation, backtesting, recommendations, broker controls, or execution APIs
are allowed.

Development remains on Mac mini M2 / macOS / Apple Silicon. The target desktop
product remains Windows-native Stark Terminal.
